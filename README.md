# UnrealActionReplicated

언리얼의 Replicated 이용시 항상 모든 Replicated 객체가 동기화 되기를 바라지 않는다.
가령 현재 나의 액션 정보를 동기화한다고 할때

걷고 있을때 필요한 동기화 정보(예:바라보는 방향 정보)
FVector dir;
공격하고 있을때 필요한 동기화 정보(예:공격방향,타겟정보)
FVector attackdir;
int targetID;

처럼 현재 나의 상태에 따라 클라에 보낼 나의 정보를 셋팅할것이다.
문제는 이러한 상태에 따른 정보가 수도없이 많이 만들어지지만 동기화가 필요한 순간 클라이언트는 모든정보를 동기화 시킨다.

가령 내가 걷고만 있어도 새로운 클라이언트가 들어오면 나의 공격방향,타겟정보는 동기화를 한다.
해당 소스는 이렇게 필요한 정보만 동기화를 시키는 구조를 잡기위한 코드가 담겨져 있다.
